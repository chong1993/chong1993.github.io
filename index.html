hexo+github搭建个人博客

hexo是目前最主流的静态博客框架，该框架基于node，因此编译效率非常高，同时借助github可以实现零成本搭建和维护，Let’s go！

环境搭建

安装node环境

在node官网下载d对应操作系统的node安装包，安装过程比较简单，就是next模式，安装完毕之后通过以下步骤检测是否安装成功：

  • win+r打开命令行窗口
  • 输入cmd回车
  • 在cmd窗口输入node -v

如果提示“node不是外部或内部命令”，则表示安装失败

安装git

git：版本控制器
进入git官网下载对应操作系统的git安装包，next模式安装

安装vscode编译器

vscode是一款强大的IDE

1226学习笔记

一.如何直接在编码器中修改桌面文件？

1.右键点击git bash here；

2.输入ls查看桌面文件；

3.输入touch 文件名.格式（例如：a.txt）；

4.输入vi 文件名.格式；

5.输入i（插入），在输入想书写的内容；

6.保存：esc键+：键+w+q键；

7.window+r键输入cmd进入程序界面，输入cd 存放路径（例如桌面：desktop cd desktop)；

注释：cd ..（释义：返回上一级）

二.如何使用hexo+node+gif搭建个人博客？

1.环境/工具

1.1、node：一款牛叉的web服务器，目前很多大型项目都使用node搭建；

1.2、特点：解析效率高，没有兼容性，比较适合io密集型的项目；

1.3版本：建议版本是6.19（比较稳定，兼容性好）；

2.如何安装

2.1、点击https://nodejs.org 下载node/vscode(多语言开发工具)/git(版本控制器)相应操作系统的安装包，安装过程可以一直按下一步；

3.如何测试nede是否安装成功

3.1、首先win键+R输入cmd回车（enter键），在打开的编码窗口中命令输入 node -v ，如果能显示node版本则安装成功，否则重新安装；

4.如何安装hexo脚手架

4.1、Window键+R输入 cmd并回车（enter键）；

4.2、在cmd回车后弹出的编码窗口中输入 npm install -g hexo-cli

注释

a、npm：依赖包管理工具
b、install：导入/安装
c、-g：全局安装

5.如何初始化一个博客项目

5.1、hexo init 项目名称(英文）；

注：init 初始化

6.如何进入项目并添加依赖包

（如果在拷贝文件时，一定要先将文件夹中node_modules文件夹删除，并重新添加依赖包）

6.1、先进入到项目主目录文件夹；

6.2、选中文件夹并点击鼠标右键git bash here ；

6.3、再弹出编码窗口后，使用npm install命令添加依赖包；

目的：将项目所需依赖包添加到本地

7.如何启动node服务器并打开博客网页

方式一：7.1、选中博客文件夹右键 git bash here；

7.2、在弹出的编码框中输入 hexo server；

7.3、在浏览器地址栏输入：localhost：4000回车浏览；

注：在打开页面时，服务器不能处在关闭状态（快捷键ctrl+c停止服务器）；

方式二：在visual studio软件中按快捷键 ctrl + ~ 键调出编码框，输入代码hexo s 按回车键；

8.如何对博客页面中的一些内容进行修改

8.1、首先打开visual syudio软件，按快捷键 ctrl + ~ 键调出编码框，按+按钮新建一个编码窗口，在新建的编码窗口中输入hexo new 文章名并按回车（例如：hexo new 我的第一篇博客）；

8.2、对文章的标题可以进行修改（title中的内容，注意—之上的内容之间均有空格键，例如：titile 1226学习笔记；### 8.3、对修改的文章，按住ctrl+s键保存后刷新博客页面，也可以在编码框中输入 hexo g并回车（编译）；

注：浏览器刷新页面快捷键 ctrl+R

三.markdown的语法

1、#符号

#+空格：有几个#号就是几级标题(例如三级标题 ###+空格)；

2、-符号

- + 空格：无序列表；

3、1.

1. + 空格：有序列表；

（例如：1. 列表项1/1. 列表项2/1. 列表项3所显示出的效果为1.列表项1/2.列表项2/3.列表项3）；

注：1.后面一定要加空格
